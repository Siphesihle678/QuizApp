Essential Concepts

Database Objects

Microsoft Access consists of four main objects that work together to create a complete database system.

‚ñ∂Tables - Data Storage Foundation

Purpose: Store data in rows (records) and columns (fields)

  • Each row represents one record (e.g., one student)
  • Each column represents one field (e.g., student name, grade)
  • Primary key uniquely identifies each record
  • Data types determine what can be stored (text, number, date, etc.)
Example: Students table with fields: StudentID (Primary Key), FirstName, LastName, Grade, Email
‚ñ∂Queries - Data Retrieval Tools

Purpose: Ask questions about your data and retrieve specific information

  • Filter data using criteria (WHERE clauses)
  • Sort results (ORDER BY)
  • Perform calculations (aggregate functions)
  • Join multiple tables to get related data
Example: "Show me all Grade 11 students who have borrowed more than 5 books"
‚ñ∂Forms - User Interface

Purpose: Provide user-friendly interface for data entry and editing

  • Easier than entering data directly in tables
  • Can include validation rules and help text
  • Show related data from multiple tables
  • Professional appearance for end users
Example: Student registration form with dropdown for grade selection and validation for email format
‚ñ∂Reports - Data Presentation

Purpose: Present data in formatted, professional output suitable for printing

  • Include calculations and summaries
  • Group and sort data logically
  • Add headers, footers, and page numbers
  • Export to PDF, Word, or Excel
Example: Monthly borrowing report showing total books borrowed per student with overdue fines

Field Properties

‚ñ∂Validation Rules

Control what data can be entered into a field:

  • >0 - Must be greater than zero
  • Between 8 And 12 - Must be between 8 and 12
  • Like "*@*.*" - Must contain @ symbol (email format)
  • Is Not Null - Field cannot be empty
‚ñ∂Input Masks

Control the format of data entry:

  • 000-000-0000 - Phone number format
  • 00/00/0000 - Date format
  • L0000000 - Student ID (letter + 7 digits)

Query Design & SQL

Basic Query Structure

SELECT [fields to display]
FROM [table name]
WHERE [conditions]
ORDER BY [sort field];

Logical Operators

AND

Both conditions must be true

Grade = "11" AND IsActive = Yes

OR

Either condition can be true

Genre = "Fiction" OR Genre = "Science"

NOT

Exclude records that match

NOT Status = "Returned"

Aggregate Functions

‚ñ∂Common Functions

COUNT()

Count number of records

COUNT(*) AS TotalStudents

SUM()

Add up numeric values

SUM(FineAmount) AS TotalFines

AVG()

Calculate average

AVG(Price) AS AveragePrice

MAX()/MIN()

Find highest/lowest value

MAX(DueDate) AS LatestDue

Database Relationships

Relationships connect related data across tables, reducing redundancy and ensuring data integrity.

Types of Relationships

One-to-One

One record in Table A relates to exactly one record in Table B

Example: Student ‚Üî Student ID Card

One-to-Many

One record in Table A relates to many records in Table B

Example: Student ‚Üî Borrowings

Many-to-Many

Many records in Table A relate to many records in Table B

Example: Students ‚Üî Books (via Borrowings)

Foreign Keys

Definition: A field in one table that references the primary key of another table

Purpose: Establish relationships and maintain referential integrity

Database Best Practices

‚úÖ Do's

  • Plan your database structure first
  • Use meaningful field names
  • Always include primary keys
  • Set up proper relationships
  • Use appropriate data types
  • Add validation rules
  • Backup your database regularly

‚ùå Don'ts

  • Store the same data in multiple places
  • Use spaces in field names
  • Forget to set up relationships
  • Use text for numeric data
  • Ignore validation rules
  • Forget to backup data
  • Create overly complex queries

Common Mistakes to Avoid

‚ñ∂Design Errors
  • Data Redundancy: Storing the same information in multiple tables
  • Poor Naming: Using unclear or inconsistent field names
  • Missing Relationships: Not connecting related tables properly
  • Wrong Data Types: Using text for numbers or dates
  • No Validation: Allowing invalid data to be entered

Summary

  • Microsoft Access has four main objects: Tables, Queries, Forms, and Reports
  • Choose appropriate data types and set up validation rules for data integrity
  • Use relationships to connect related data and reduce redundancy
  • Design queries carefully using logical operators and aggregate functions
  • Create user-friendly forms for data entry and professional reports for output
  • Import/export data to work with other applications
  • Follow best practices to create efficient, maintainable databases
