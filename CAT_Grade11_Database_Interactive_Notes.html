A database is an organized collection of data stored electronically. Microsoft Access is a database management system that allows you to store, retrieve, and manage information efficiently.

  • Hospital patient records
  • Online store inventory

üîç Queries

What are Queries?
Queries are questions you ask your database. They allow you to retrieve specific data from one or more tables based on criteria you specify.
Query Examples:
  • "Show me all students in Grade 11"
  • "Find books that cost more than R100"
  • "List overdue books and their borrowers"
  • "Count how many books each student has borrowed"
Logical Operators
Logical operators help you create more complex queries by combining multiple conditions.
Operator Meaning Example
AND Both conditions must be true Grade = "11" AND IsActive = Yes
OR Either condition can be true Genre = "Fiction" OR Genre = "Science"
NOT Exclude records that match NOT Status = "Returned"
Query Builder Demo
Sample Query:
SELECT FirstName, LastName, Grade
FROM Students
WHERE Grade = "11" AND IsActive = Yes
ORDER BY LastName, FirstName;
This query finds: All active Grade 11 students, sorted by last name then first name.
üí° Query Tips
  • Always test your queries with sample data first
  • Use meaningful names for your queries
  • Start simple and add complexity gradually
  • Use the Query Design view for visual query building

üìù Forms

What are Forms?
Forms provide a user-friendly interface for entering, editing, and viewing data. They make data entry more efficient and reduce errors.
Form Benefits:
  • Easier data entry than tables
  • Better data validation
  • Professional appearance
  • Can include instructions and help text
  • Can show related data from multiple tables
Form Controls
Forms use various controls to display and collect data effectively.
Control Purpose Example
Text Box Enter text or numbers Student name, phone number
Combo Box Select from a list Grade selection (8, 9, 10, 11, 12)
Check Box Yes/No selection Active status, available
Date Picker Select dates Birth date, due date
Button Perform actions Save, delete, search
üí° Form Design Tips

  • Use clear, descriptive labels
  • Include validation messages
  • Make the form visually appealing
  • Test the form with real users

‚úÖ Data Validation

What is Data Validation?
Data validation ensures that only correct, complete, and reasonable data is entered into your database. It helps maintain data integrity and quality.
Validation Examples:
  • Email addresses must contain @ symbol
  • Phone numbers must be 10 digits
  • Grades must be between 8 and 12
  • Prices must be greater than 0
  • Required fields cannot be empty
Validation Methods
Access provides several ways to validate data at different levels.
Method Where Applied Example
Field Properties Table level Required = Yes, Field Size = 50
Validation Rules Field level >0 AND <=100
Input Masks Field level Phone number format
Form Validation Form level Before Update event
Validation Rule Examples
Common Validation Rules:
‚Ä¢ Numbers: >0 AND <=100
‚Ä¢ Dates: >=#1/1/2000# AND <=Date()
‚Ä¢ Text: Like "*@*.*" (email format)
‚Ä¢ Required: Is Not Null
‚Ä¢ Range: Between 8 And 12
‚ö†Ô∏è Validation Best Practices
  • Always provide clear error messages
  • Test validation rules thoroughly
  • Don't make rules too restrictive
  • Consider user experience
  • Document your validation rules
